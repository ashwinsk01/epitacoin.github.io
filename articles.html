Building the EFOBE


An NP-Complete problem shares some properties with reverse hashing: no polynomial time algorithm is known to solve them, but if a solution is found,then the validity of that solution can be checked in polynomial time. This fact induced the design of EFOBEwhich is an acronym for Epicoin’s Free and OpenBlockchain of Epicness.Here follows a concrete example of the use of the EFOBE : Some university has a problem to solve. It would post a request to the block-chain containing the name 3 of the problem and input parameters for that problem.Miners seeking to gain units of Epicoin, or to decide on the next problem to solve, would then compete to solve that university’s problem and initialize anew block with the input parameters.The miner would identify the problem using its name. It would then choose the appropriate method to solve that problem and would call it with the given input parameters.The problem being NP-complete, a miner solving it first would push the block to the network. That block would contain the solution, an identifier which could be a quickly-computed hash, the input parameters and the identifier ofthe preceding block.The only thing left for the Epicoin to be complete is to clarify the incentives to mine.
